Instalação

Coloque o Hivekeep no ar.

Um comando faz tudo. Quando estiver no ar, a Queenie configura o resto conversando com você, então não há arquivos de config para editar.

Recomendado · o jeito mais simples

Cole uma linha. Só isso.

Rode isto em um terminal na sua máquina Linux ou macOS. Ele instala tudo para você.

curl -fsSL https://raw.githubusercontent.com/MarlBurroW/hivekeep/main/install.sh | bash

Quando terminar, abra no navegador o link que ele imprime. A Queenie faz o resto.

Opcional

Quer uma porta customizada, um domínio ou Docker?

Pule esta parte se a linha única acima resolve para você. Caso contrário, responda algumas perguntas e geramos o comando exato para o seu setup.

Instalarshell
curl -fsSL https://raw.githubusercontent.com/MarlBurroW/hivekeep/main/install.sh | bash

Sua chave de criptografia é cuidada para você. O instalador a gera e salva automaticamente em $DATA_DIR/.encryption-key para que seus segredos sobrevivam a restarts. Faça backup desse arquivo junto com seu banco de dados. (Fixe uma ENCRYPTION_KEY com o toggle avançado acima se preferir gerenciá-la você mesmo.)

Se a instalação falhar
  • port already in use / EADDRINUSE: a porta 3000 está ocupada. Mude o campo Porta acima e rode de novo.
  • Windows: o instalador é só para Linux e macOS. Rode-o dentro do WSL2, ou use o Docker Desktop.
  • O download ou o clone trava: confira se a máquina alcança github.com e bun.sh via HTTPS (um proxy pode estar bloqueando).

Abra http://localhost:3000 no seu navegador. A Queenie guia você pelo resto (conta de admin, seu primeiro provedor de IA, seus primeiros agentes). Nenhum arquivo de config para editar.

Antes de começar

O que a linha única precisa.

A maioria das máquinas já tem tudo isso.

Você pode rodar sudo
O instalador precisa dele para instalar pacotes de sistema que faltarem (git, curl, unzip).
Uns 500 MB livres em disco
Espaço para clonar e compilar. Com 1 GB ou mais fica confortável.
Uma máquina 64 bits
x86_64 ou ARM64. 32 bits (Raspberry Pi mais antigos) não é suportado.
HTTPS de saída
Ele baixa de github.com e bun.sh, então esses endereços precisam estar acessíveis.
openssl presente
Usado para gerar as chaves. Vem com praticamente todo Linux e macOS.

No Windows? O instalador nativo não roda direto no Windows. Use o WSL2 (rode a linha única dentro da sua distro Linux) ou o Docker Desktop.

Compare

Nativo, Docker ou do código-fonte?

Os três rodam exatamente o mesmo app. A diferença está em como ele vive na máquina.

Recomendado

Nativo (install.sh)

Escolha este se você só quer que funcione. Um comando, na sua própria máquina Linux ou macOS.

  • Um comando, compila localmente, sem imagem para publicar ou baixar
  • Salva sua chave de criptografia automaticamente, os segredos persistem
  • Roda como serviço (systemd / launchd), atualização automática com rollback
  • Os agentes são donos da máquina: instalam ferramentas e dependências, acesso direto ao hardware
  • Modifica o sistema host (de propósito)
  • Só Linux e macOS (Windows via WSL2)
Container

Docker

Escolha este se você já vive no Docker e quer um appliance limpo e isolado.

  • Totalmente isolado, zero poluição no host
  • Roda onde o Docker rodar, inclusive no Windows
  • A imagem publicada ainda não está disponível, prefira o nativo por enquanto
  • Ferramentas e binários instalados pelos agentes não sobrevivem a um restart
  • Você precisa persistir o volume de dados ou perde seus segredos
Do código-fonte

Manual

Escolha este se você quer ler o código, mexer nele ou rodar do seu próprio jeito.

  • Controle total: você mesmo clona, compila e roda com Bun
  • Ideal para quem contribui e desenvolve
  • Sem serviço, sem atualização automática, você monta isso por conta própria
  • Bun, build, migrações e chave de criptografia ficam na sua mão
