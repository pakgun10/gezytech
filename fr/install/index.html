Installation

Mets Hivekeep en route.

Une commande fait tout. Une fois lancé, Queenie configure le reste en discutant avec toi : aucun fichier de config à éditer.

Recommandé · la façon la plus simple

Colle une ligne. C'est tout.

Lance ceci dans un terminal sur ta machine Linux ou macOS. Tout s'installe pour toi.

curl -fsSL https://raw.githubusercontent.com/MarlBurroW/hivekeep/main/install.sh | bash

Quand c'est terminé, ouvre dans ton navigateur le lien affiché. Queenie fait le reste.

Passe cette étape si la ligne ci-dessus te suffit. Sinon, réponds à deux ou trois questions et on génère la commande exacte pour ta configuration.

Installershell
curl -fsSL https://raw.githubusercontent.com/MarlBurroW/hivekeep/main/install.sh | bash

Ta clé de chiffrement est gérée pour toi. L'installateur la génère et l'enregistre automatiquement dans $DATA_DIR/.encryption-key : tes secrets survivent aux redémarrages. Sauvegarde ce fichier avec ta base de données. (Fixe une ENCRYPTION_KEY avec l'option avancée ci-dessus si tu préfères la gérer toi-même.)

Si l'installation échoue
  • port already in use / EADDRINUSE : le port 3000 est pris. Change le champ Port ci-dessus et relance.
  • Windows : l'installateur est Linux et macOS uniquement. Lance-le dans WSL2, ou utilise Docker Desktop.
  • Le téléchargement ou le clone bloque : vérifie que la machine peut joindre github.com et bun.sh en HTTPS (un proxy peut les bloquer).

Ouvre http://localhost:3000 dans ton navigateur. Queenie te guide pour la suite (compte admin, ton premier fournisseur d'IA, tes premiers agents). Aucun fichier de config à éditer.

Avant de commencer

Ce dont la commande unique a besoin.

La plupart des machines ont déjà tout ça.

Tu peux lancer sudo
L'installateur en a besoin pour ajouter les paquets système manquants (git, curl, unzip).
Environ 500 Mo de disque libre
De quoi cloner et builder. 1 Go ou plus, c'est confortable.
Une machine 64 bits
x86_64 ou ARM64. Le 32 bits (anciens Raspberry Pi) n'est pas supporté.
HTTPS sortant
Le script télécharge depuis github.com et bun.sh : ces domaines doivent être joignables.
openssl présent
Utilisé pour générer les clés. Il est livré avec presque tous les Linux et macOS.

Sous Windows ? L'installateur natif ne tourne pas directement sous Windows. Utilise WSL2 (lance la commande dans ta distro Linux) ou Docker Desktop.

Comparer

Natif, Docker ou depuis les sources ?

Les trois font tourner exactement la même app. Ils diffèrent par la façon dont elle vit sur la machine.

Recommandé

Natif (install.sh)

Choisis ça si tu veux juste que ça marche. Une commande, sur ta propre machine Linux ou macOS.

  • Une commande, build local, aucune image à publier ni à télécharger
  • Sauvegarde automatiquement ta clé de chiffrement, les secrets persistent
  • Tourne en service (systemd / launchd), mise à jour auto avec rollback
  • Les agents sont chez eux : installation d'outils & dépendances, accès direct au matériel
  • Modifie le système hôte (c'est voulu)
  • Linux & macOS uniquement (Windows via WSL2)
Conteneur

Docker

Choisis ça si tu vis déjà dans Docker et veux une appliance propre et isolée.

  • Totalement isolé, zéro pollution de l'hôte
  • Tourne partout où Docker tourne, Windows compris
  • L'image publiée n'est pas encore disponible, préfère le natif pour l'instant
  • Les outils & binaires installés par les agents ne survivent pas à un redémarrage
  • Il faut persister le volume de données, sinon tu perds tes secrets
Depuis les sources

Manuel

Choisis ça si tu veux lire le code, bidouiller, ou tout faire à ta façon.

  • Contrôle total : clone, build et lancement avec Bun par toi-même
  • Idéal pour les contributeurs et le développement
  • Pas de service, pas de mise à jour auto : à toi de les câbler
  • Bun, build, migrations et clé de chiffrement sont à gérer à la main
