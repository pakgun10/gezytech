安装

让 Hivekeep 跑起来。

一条命令搞定一切。启动之后，Queenie 会通过聊天帮你完成其余配置，没有任何配置文件需要编辑。

推荐 · 最简单的方式

粘贴一行，就这么简单。

在你的 Linux 或 macOS 机器的终端里运行这条命令，它会为你装好一切。

curl -fsSL https://raw.githubusercontent.com/MarlBurroW/hivekeep/main/install.sh | bash

完成后，在浏览器中打开它输出的链接。剩下的交给 Queenie。

安装shell
curl -fsSL https://raw.githubusercontent.com/MarlBurroW/hivekeep/main/install.sh | bash

加密密钥已自动为你处理。安装器会自动生成密钥并保存在 $DATA_DIR/.encryption-key，让你的机密在重启后依然可用。请把该文件和数据库一起备份。（如果你更想自己管理，可通过上方的高级开关固定一个 ENCRYPTION_KEY。）

如果安装失败
  • port already in use / EADDRINUSE：端口 3000 已被占用。修改上方的端口字段后重新运行。
  • Windows：安装器仅支持 Linux 和 macOS。请在 WSL2 中运行，或使用 Docker Desktop。
  • 下载或克隆卡住：请确认机器能通过 HTTPS 访问 github.com 和 bun.sh（可能被代理拦截）。

在浏览器中打开 http://localhost:3000。Queenie 会引导你完成其余步骤（管理员账号、第一个 AI 服务商、第一批智能体）。没有任何配置文件需要编辑。
高级与其他选项 系统要求，以及 Docker、原生与源码安装的对比
开始之前

一行命令需要什么。

大多数机器早已具备这一切。

可以运行 sudo
安装器需要它来补齐缺失的系统包（git、curl、unzip）。
约 500 MB 可用磁盘空间
用于克隆和构建。1 GB 以上会更宽裕。
64 位机器
x86_64 或 ARM64。不支持 32 位（较老的树莓派）。
可出站 HTTPS
需要从 github.com 和 bun.sh 下载，这两个地址必须可达。
已安装 openssl
用于生成密钥。几乎所有 Linux 和 macOS 都自带。

在 Windows 上？原生安装器无法直接在 Windows 运行。请使用 WSL2（在你的 Linux 发行版里运行那行命令）或 Docker Desktop。

对比

原生、Docker，还是源码？

三种方式运行的是完全相同的应用，区别只在它如何驻留在机器上。

推荐

原生（install.sh）

只想让它直接能用？选这个。一条命令，跑在你自己的 Linux 或 macOS 机器上。

  • 一条命令，本地构建，无需发布或拉取镜像
  • 自动保存加密密钥，机密数据持久保留
  • 以服务运行（systemd / launchd），自动更新并支持回滚
  • 智能体拥有整台机器：可安装工具与依赖，直接访问硬件
  • 会修改宿主系统（这是设计使然）
  • 仅限 Linux 与 macOS（Windows 需通过 WSL2）
容器

Docker

如果你已经常驻 Docker，想要一个干净、沙箱化的开箱设备，就选这个。

  • 完全隔离，零宿主污染
  • 凡是能跑 Docker 的地方都能跑，包括 Windows
  • 官方镜像暂未发布，目前请优先选择原生安装
  • 智能体安装的工具和二进制在重启后不会保留
  • 必须持久化数据卷，否则会丢失全部机密
源码

手动

想读代码、改代码，或按自己的方式运行？选这个。

  • 完全掌控：自己用 Bun 克隆、构建、运行
  • 最适合贡献者和开发场景
  • 没有服务托管，没有自动更新，需要自己搭建
  • Bun、构建、迁移和加密密钥都要手动处理
