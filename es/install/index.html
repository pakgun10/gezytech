Instalación

Pon Hivekeep en marcha.

Un solo comando lo hace todo. Una vez arrancado, Queenie configura el resto charlando contigo, así que no hay archivos de configuración que editar.

Recomendado · la forma más simple

Pega una línea. Eso es todo.

Ejecuta esto en una terminal de tu máquina Linux o macOS. Lo instala todo por ti.

curl -fsSL https://raw.githubusercontent.com/MarlBurroW/hivekeep/main/install.sh | bash

Cuando termine, abre en tu navegador el enlace que imprime. Queenie hace el resto.

Opcional

¿Quieres un puerto personalizado, un dominio o Docker?

Sáltate esto si con la línea de arriba te basta. Si no, responde un par de preguntas y generaremos el comando exacto para tu configuración.

Instalarshell
curl -fsSL https://raw.githubusercontent.com/MarlBurroW/hivekeep/main/install.sh | bash

Tu clave de cifrado se gestiona por ti. El instalador la genera y la guarda automáticamente en $DATA_DIR/.encryption-key para que tus secretos sobrevivan a los reinicios. Haz copia de seguridad de ese archivo junto con tu base de datos. (Fija una ENCRYPTION_KEY con el interruptor avanzado de arriba si prefieres gestionarla tú mismo.)

Si la instalación falla
  • port already in use / EADDRINUSE: el puerto 3000 está ocupado. Cambia el campo Puerto de arriba y vuelve a ejecutar.
  • Windows: el instalador es solo para Linux y macOS. Ejecútalo dentro de WSL2, o usa Docker Desktop.
  • ¿La descarga o el clon se quedan colgados? Asegúrate de que la máquina llega a github.com y bun.sh por HTTPS (puede que un proxy los esté bloqueando).

Abre http://localhost:3000 en tu navegador. Queenie te guía con el resto (cuenta de administrador, tu primer proveedor de IA, tus primeros agentes). Sin archivos de configuración que editar.

Antes de empezar

Qué necesita el comando de una línea.

La mayoría de las máquinas ya tienen todo esto.

Puedes ejecutar sudo
El instalador lo necesita para instalar los paquetes del sistema que falten (git, curl, unzip).
Unos 500 MB de disco libres
Espacio para clonar y compilar. Con 1 GB o más vas sobrado.
Una máquina de 64 bits
x86_64 o ARM64. 32 bits (Raspberry Pi antiguas) no está soportado.
HTTPS de salida
Descarga desde github.com y bun.sh, así que ambos deben ser accesibles.
openssl presente
Se usa para generar claves. Viene con casi cualquier Linux y macOS.

¿En Windows? El instalador nativo no funciona directamente en Windows. Usa WSL2 (ejecuta el comando de una línea dentro de tu distro Linux) o Docker Desktop.

Compara

¿Nativo, Docker o desde el código fuente?

Las tres opciones ejecutan exactamente la misma app. Se diferencian en cómo vive en la máquina.

Recomendado

Nativo (install.sh)

Elige esta opción si solo quieres que funcione. Un comando, en tu propia máquina Linux o macOS.

  • Un comando, compila en local, sin imagen que publicar ni descargar
  • Guarda tu clave de cifrado automáticamente, los secretos persisten
  • Funciona como servicio (systemd / launchd), autoactualización con rollback
  • Los agentes son dueños de la máquina: instalan herramientas y dependencias, acceso directo al hardware
  • Modifica el sistema anfitrión (por diseño)
  • Solo Linux y macOS (Windows vía WSL2)
Contenedor

Docker

Elige esta opción si ya vives en Docker y quieres un appliance limpio y aislado.

  • Totalmente aislado, cero contaminación del host
  • Funciona donde funcione Docker, incluido Windows
  • La imagen publicada aún no está disponible, por ahora prefiere el nativo
  • Las herramientas y binarios que instalan los agentes no sobreviven a un reinicio
  • Debes persistir el volumen de datos o perderás tus secretos
Desde el código

Manual

Elige esta opción si quieres leer el código, trastear con él o ejecutarlo a tu manera.

  • Control total: clona, compila y ejecuta con Bun tú mismo
  • Lo mejor para contribuidores y desarrollo
  • Sin servicio ni autoactualización, eso lo montas tú
  • Bun, la build, las migraciones y la clave de cifrado corren de tu cuenta
